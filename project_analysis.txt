# Ανάλυση Project "Homease"

Αυτή η ανάλυση εξετάζει το project "Homease" σε σχέση με τις προδιαγραφές που τέθηκαν.

## Backend

### 1. Τουλάχιστον 10 διαθέσιμα routes/API calls

Το backend του Homease διαθέτει πάνω από 20 routes. Ενδεικτικά:

- `POST /api/v1/auth/login`
- `POST /api/v1/automations`
- `GET /api/v1/devices/available`
- `DELETE /api/v1/devices/:deviceId`
- `PUT /api/v1/devices/:deviceId`
- `GET /api/v1/devices/:deviceId/status`
- `POST /api/v1/devices/:deviceId/action`
- `GET /api/v1/houses`
- `POST /api/v1/houses`
- `DELETE /api/v1/houses/:houseId`
- `GET /api/v1/houses/:houseId/rooms`
- `POST /api/v1/houses/:houseId/rooms`
- `DELETE /api/v1/rooms/:roomId`
- `PUT /api/v1/rooms/:roomId/temperature`
- `PUT /api/v1/rooms/:roomId/lighting`
- `GET /api/v1/rooms/:roomId/devices`
- `POST /api/v1/rooms/:roomId/devices`
- `POST /api/v1/scenarios`
- `PUT /api/v1/security/:deviceId/state`
- `POST /api/v1/users`
- `GET /api/v1/users/:userId/statistics`
- `POST /api/v1/users/:userId/notifications`
- `PUT /api/v1/users/:userId/home`
- `POST /api/v1/users/:userId/widgets`
- `GET /api/v1/users/me`

**=> Η προδιαγραφή καλύπτεται.**

### 2. Τουλάχιστον ένα route τύπου GET, POST, PUT και DELETE

Για το resource `devices`, υπάρχουν τα παρακάτω routes:

- **GET**: `/api/v1/devices/available`
- **POST**: `/api/v1/devices/:deviceId/action`
- **PUT**: `/api/v1/devices/:deviceId`
- **DELETE**: `/api/v1/devices/:deviceId`

**=> Η προδιαγραφή καλύπτεται.**

### 3. Τουλάχιστον 3 διαφορετικές οντότητες

Το project περιλαμβάνει τις εξής οντότητες (μοντέλα):

- **User**: Ο χρήστης της εφαρμογής.
- **House**: Το σπίτι του χρήστη.
- **Room**: Ένα δωμάτιο μέσα στο σπίτι.
- **Device**: Μια έξυπνη συσκευή μέσα σε ένα δωμάτιο.
- **Automation**: Ένας κανόνας αυτοματισμού.
- **Scenario**: Ένα σενάριο που εκτελεί πολλές ενέργειες.

Οι οντότητες αυτές αλληλεπιδρούν μεταξύ τους. Για παράδειγμα, ένας `User` έχει `House`, ένα `House` έχει `Room`, και ένα `Room` έχει `Device`.

**=> Η προδιαγραφή καλύπτεται.**

### 4. Mock δεδομένα

Το project χρησιμοποιεί hardcoded mock δεδομένα, τα οποία βρίσκονται στο αρχείο `backend/config/mockData.js`. Αυτά τα δεδομένα φορτώνονται κατά την εκκίνηση του server και χρησιμοποιούνται από τις υπηρεσίες (services) για την εξυπηρέτηση των API calls. Υπάρχει επίσης η υποδομή για προαιρετική σύνδεση με βάση δεδομένων MongoDB.

**=> Η προδιαγραφή καλύπτεται.**

## Frontend

### 1. Τουλάχιστον 4 οθόνες (screens)

Το frontend αποτελείται από τις παρακάτω οθόνες (pages):

- `AddHouseRoomPage.jsx`
- `AutomationPage.jsx`
- `DashboardPage.jsx`
- `LightingControlPage.jsx`
- `LoginPage.jsx`
- `ManageDevicesPage.jsx`

**=> Η προδιαγραφή καλύπτεται.**

### 2. Τουλάχιστον 5 backend endpoints καλούνται

Το frontend καλεί πολλά endpoints του backend. Ενδεικτικά:

- `POST /auth/login` (για τη σύνδεση του χρήστη)
- `GET /users/me` (για την ανάκτηση των στοιχείων του συνδεδεμένου χρήστη)
- `GET /houses` (για την ανάκτηση των σπιτιών του χρήστη)
- `GET /houses/:houseId/rooms` (για την ανάκτηση των δωματίων ενός σπιτιού)
- `GET /rooms/:roomId/devices` (για την ανάκτηση των συσκευών ενός δωματίου)
- `GET /devices/available` (για την ανάκτηση των διαθέσιμων συσκευών)
- `POST /rooms/:roomId/devices` (για την προσθήκη μιας συσκευής σε ένα δωμάτιο)
- `DELETE /devices/:deviceId` (για τη διαγραφή μιας συσκευής)

**=> Η προδιαγραφή καλύπτεται.**
